## 🎓 Learning Path

### For a New Agent (First Time)
1. **15 min**: Read [AGENT_CONTEXT.md - Project Overview](./AGENT_CONTEXT.md#project-overview) & [Architecture](./AGENT_CONTEXT.md#architecture-overview)
2. **10 min**: Skim [AGENT_CONTEXT.md - Known Issues & Fixes](./AGENT_CONTEXT.md#known-issues--fixes)
3. **5 min**: Check [AGENT_CONTEXT.md - File Organization](./AGENT_CONTEXT.md#file-organization)
4. **20 min**: Follow [Development Setup](./AGENT_CONTEXT.md#development-setup)
5. **10 min**: Run `npm run test:unit` and verify it works

**Total: ~60 minutes to be productive**

### For Debugging an Issue
1. Check [AGENT_CONTEXT.md - Known Issues & Fixes](./AGENT_CONTEXT.md#known-issues--fixes)
2. Look at error message in browser console (F12)
3. Search [AGENT_CONTEXT.md - Troubleshooting](./AGENT_CONTEXT.md#troubleshooting) section
4. Check [FIREBASE_FIXES_APPLIED.md](./FIREBASE_FIXES_APPLIED.md) if Firebase-related
5. Review relevant code file mentioned in error

### For Making Changes
1. Read [AGENT_CONTEXT.md - Critical Components](./AGENT_CONTEXT.md#critical-components-to-know) for relevant component
2. Check existing tests in `src/test/`
3. Make changes
4. Run `npm test` (watch mode)
5. Verify tests pass: `npm run test:ci`
6. Commit with descriptive message

### For Deploying
1. Ensure all tests pass: `npm run test:ci`
2. Ensure build succeeds: `npm run build`
3. Review [DEPLOYMENT_STATUS.md](./DEPLOYMENT_STATUS.md)
4. Push to Gitea: `git push origin main`
5. Wait for Watchtower (5-10 minutes)
6. Verify using [Deployment - Verification Checklist](./AGENT_CONTEXT.md#verification-checklist)
